**1. Parse Và Validate**

Transform: `ParseValidateDoFn` ở [transforms.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/transforms.py:16).

Logic dự kiến:

- Decode Pub/Sub message bytes sang UTF-8.
- Parse JSON.
- Hỗ trợ alias:
  - `event_id` hoặc `id`
  - `event_timestamp` hoặc `created_at`
- Nếu không có `ingested_at`, lấy bằng `event_timestamp`.
- Check required fields:
  - `event_id`
  - `session_id`
  - `event_type`
  - `event_timestamp`
  - `ingested_at`
- Validate `event_type` thuộc tập ở [transforms.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/transforms.py:12):

```text
home, department, category, product, cart, purchase, cancel, return
```

- Parse timestamp về UTC bằng `parse_iso8601`.
- Chuẩn hóa timestamp string bằng `to_timestamp_string`.
- Tạo `event_date` từ ngày của `event_timestamp`.
- Gán Beam event-time bằng `TimestampedValue(row, event_ts.timestamp())`.

Nếu lỗi, record đi vào dead-letter với:

- `raw_message`
- `error_message`
- `failed_at`

Điểm hiện trạng cần chú ý: [transforms.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/transforms.py:28) đang gọi `json.loads(raw_text)` nhưng file chưa `import json`. Nếu chạy đúng code hiện tại, parse sẽ ném `NameError: name 'json' is not defined`, khiến message bị đưa vào dead-letter thay vì đi tiếp pipeline.

**3. Deduplicate Theo Event ID**

Sau parse valid, pipeline key theo `event_id`:

```python
(row["event_id"], row)
```

ở [pipeline.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/pipeline.py:74).

Transform: `DeduplicateEventsDoFn` ở [transforms.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/transforms.py:116).

Logic:

- Dùng Beam user state `seen`.
- Nếu `event_id` chưa thấy:
  - ghi `seen = True`
  - set watermark timer tại `event_timestamp + ttl_seconds`
  - emit row
- Nếu đã thấy:
  - tăng metric `duplicate_events`
  - bỏ qua row
- Khi timer fire, clear state.

TTL mặc định là `dedup_ttl_minutes * 60`, tức 60 phút nếu dùng default.

Lưu ý: đây là runtime dedup, không phải exactly-once tuyệt đối. Nếu job restart hoặc duplicate đến ngoài TTL, duplicate vẫn có thể lọt xuống BigQuery.

**4. Enrichment Product Và URI**

Transform: `EnrichEventDoFn` ở [transforms.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/transforms.py:89).

Trước đó pipeline load product dimension một lần khi job start ở [pipeline.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/pipeline.py:52):

- Load từ BigQuery bằng `load_product_dimension_from_bigquery`.
- Nếu lỗi và có CSV fallback thì đọc CSV.
- Đưa vào Beam side input bằng `AsSingleton`.

Đây là snapshot lookup, không refresh định kỳ. Muốn cập nhật product metadata thì restart job.

URI parsing nằm ở [utils.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/utils.py:79):

- URI rỗng/null: `page_type = unknown`, `product_id = None`
- URI `/`: `page_type = home`
- URI `/product/<id>`: `page_type = product`, `product_id = <id>`
- URI bắt đầu bằng `cancel` hoặc `return`: đổi `page_type = post_purchase`
- Các URI khác lấy segment đầu làm `page_type`

Enriched fields được thêm:

- `page_type`
- `product_id`
- `product_category`
- `product_department`
- `product_name`
- `is_conversion = event_type == "purchase"`
- `processing_time = utc_now()`
- `event_lag_seconds = max(0, ingested_at - event_timestamp)`

**7. Sessionization**

Nhánh session bắt đầu ở [pipeline.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/pipeline.py:125):

- Key theo `session_id`.
- Window bằng Beam session window:

```python

```

Default gap là 30 phút.

Sau `GroupByKey`, pipeline gọi `build_session_metric` ở [transforms.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/transforms.py:197).

Logic session metric:

- Convert rows thành list.
- Nếu session có hơn `MAX_EVENTS_PER_SESSION = 5000`, log warning và cap còn 5000 event.
- Sort event theo `event_timestamp`.
- Lấy:
  - `session_start` từ event đầu
  - `session_end` từ event cuối
  - `session_duration_seconds`
- Đếm category để lấy `top_category`.
- Đếm:
  - `cart_count`
  - `purchase_count`
  - `pageview_count`
  - `product_view_count`
- Tạo funnel flags:
  - `saw_home`
  - `saw_product`
  - `added_to_cart`
  - `purchased`

**Rủi Ro Hiện Tại**

Điểm lớn nhất là thiếu `import json` trong [transforms.py](D:/myBK/HK252/DWH/BTL-DWH/src/clickstream/pipeline/transforms.py:1), làm parse JSON lỗi ngay nếu code chạy như hiện tại.

Ngoài ra:

- Các bảng `events_raw`, `events_5m`, `session_metrics` là append-oriented, đọc trực tiếp có thể double count.
- Runtime dedup không đảm bảo exactly-once tuyệt đối.
- Product dimension là snapshot tại lúc job start, có thể stale.
- Session aggregation dùng `GroupByKey`, nên session quá lớn vẫn phải materialize trước khi cap.
