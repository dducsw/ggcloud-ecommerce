Dưới đây là luồng clickstream hiện tại theo trục **data source → processing → storage → serving**, chỉ tập trung vào **logic** và **abstract code**.

**1. Data Source: Datagen/Event Publisher**

Nguồn dữ liệu clickstream được sinh từ datagen, sau đó publisher đóng gói mỗi event thành JSON và publish lên Pub/Sub topic.

Logic payload:

```python
payload = {
    "event_id": event.id,
    "user_id": event.user_id,
    "sequence_number": event.sequence_number,
    "session_id": event.session_id,
    "ip_address": event.ip_address,
    "city": event.city,
    "state": event.state,
    "postal_code": event.postal_code,
    "browser": event.browser,
    "traffic_source": event.traffic_source,
    "uri": event.uri,
    "event_type": event.event_type,
    "event_timestamp": event.created_at,
    "ingested_at": now_utc(),
}
```

Sau đó:

```python
publish_to_pubsub(
    topic="clickstream_topic",
    message=json.dumps(payload)
)
```

Về mặt nghiệp vụ, mỗi event mô tả một hành vi người dùng trong session: vào home, xem department/category/product, add cart, purchase, cancel, return.

**2. Ingestion: Pub/Sub → Beam Streaming Pipeline**

Apache Beam/Dataflow đọc stream từ Pub/Sub subscription.

Abstract code:

```python
messages = pipeline | ReadFromPubSub(subscription=events_subscription)
```

Pipeline chạy streaming mode, dùng event-time thay vì processing-time cho các nhánh window/session.

**3. Parse + Validate**

Mỗi message bytes được decode, parse JSON, validate field bắt buộc, chuẩn hóa timestamp và gán event-time.

Logic:

```python
def parse_validate(message):
    try:
        raw_text = message.decode("utf-8")
        payload = json.loads(raw_text)

        event_id = payload.get("event_id") or payload.get("id")
        event_timestamp = payload.get("event_timestamp") or payload.get("created_at")
        ingested_at = payload.get("ingested_at") or event_timestamp

        require(event_id)
        require(payload["session_id"])
        require(payload["event_type"])
        require(event_timestamp)
        require(ingested_at)

        if payload["event_type"] not in VALID_EVENT_TYPES:
            raise InvalidEventType

        event_ts = parse_iso8601(event_timestamp)
        ingested_ts = parse_iso8601(ingested_at)

        row = {
            "event_id": int(event_id),
            "user_id": nullable_int(payload.get("user_id")),
            "sequence_number": nullable_int(payload.get("sequence_number")),
            "session_id": str(payload["session_id"]),
            "ip_address": payload.get("ip_address"),
            "city": payload.get("city"),
            "state": payload.get("state"),
            "postal_code": payload.get("postal_code"),
            "browser": payload.get("browser"),
            "traffic_source": payload.get("traffic_source"),
            "uri": payload.get("uri"),
            "event_type": payload["event_type"],
            "event_timestamp": to_utc_string(event_ts),
            "event_date": event_ts.date(),
            "ingested_at": to_utc_string(ingested_ts),
        }

        emit_with_event_time(row, event_ts)

    except Exception as error:
        emit_deadletter(raw_message=message, error_message=str(error))
```

Tập event type hợp lệ:

```python
VALID_EVENT_TYPES = {
    "home",
    "department",
    "category",
    "product",
    "cart",
    "purchase",
    "cancel",
    "return",
}
```

Kết quả sau bước này tách thành 2 nhánh:

```text
valid events      -> xử lý tiếp
invalid events    -> dead-letter
```

Các message lỗi được ghi vào BigQuery table `events_deadletter`.

Logic:

```python
deadletter_row = {
    "raw_message": raw_message,
    "error_message": error_message,
    "failed_at": now_utc(),
}
```

Ghi:

```python
write_bigquery(
    table="events_deadletter",
    rows=deadletter_rows,
    mode="append"
)
```

Mục đích: pipeline không bị chết vì dữ liệu bẩn, đồng thời vẫn có nơi audit lỗi input.

**5. Deduplicate Theo Event ID**

Các event valid được key theo `event_id`, sau đó dedup bằng Beam state.

Logic:

```python
events_by_id = valid_events.map(lambda row: (row["event_id"], row))
```

Dedup abstract:

```python
def deduplicate(event_id, row, state):
    if state.seen == True:
        metric("duplicate_events").inc()
        drop(row)
    else:
        state.seen = True
        set_timer(event_time(row) + dedup_ttl)
        emit(row)
```

Khi timer hết hạn:

```python
def on_timer():
    state.clear()
```

Ý nghĩa:

- Giảm duplicate do Pub/Sub redelivery hoặc publisher gửi lặp.
- Dedup chỉ có hiệu lực trong TTL, mặc định 60 phút.
- Đây không phải exactly-once tuyệt đối, nên serving layer vẫn có view dedup ở BigQuery.

**6. Product Dimension Lookup**

Trước khi pipeline xử lý stream, job load product dimension một lần lúc startup.

Logic:

```python
try:

except:
    product_map = load_from_csv(products_csv)
```

Product map có dạng:

```python
product_map = {
    product_id: {

```

Sau đó đưa vào Beam side input:

**7. Enrichment**

Sau dedup, event được enrich bằng URI parsing và product lookup.

URI parsing logic:

```python
def parse_uri(uri):
    if uri is None or uri == "":
        return page_type="unknown", product_id=None

    parts = uri.strip("/").split("/")

    if parts is empty:
        return page_type="home", product_id=None

    page_type = parts[0]

    if page_type == "product" and parts[1].isdigit():
        product_id = int(parts[1])

    elif page_type in {"cancel", "return"}:
        page_type = "post_purchase"
        product_id = None

    return page_type, product_id
```

Enrich logic:

```python
def enrich(row, product_map):
    page_type, product_id = parse_uri(row["uri"])

    product = product_map.get(product_id)

    enriched = row.copy()
    enriched["page_type"] = page_type
    enriched["product_id"] = product_id
    enriched["product_category"] = product["category"] if product else None
    enriched["product_department"] = product["department"] if product else None
    enriched["product_name"] = product["name"] if product else None

    enriched["is_conversion"] = row["event_type"] == "purchase"
    enriched["processing_time"] = now_utc()
    enriched["event_lag_seconds"] = max(
        0,
        row["ingested_at"] - row["event_timestamp"]
    )

    emit(enriched)
```

Sau bước này, stream chính trở thành `enriched_events`.

**8. Raw Event Sink**

Toàn bộ event đã parse, dedup, enrich được ghi append vào `events_raw`.

Abstract:

```python
write_bigquery(
    table="events_raw",
    rows=enriched_events,
    partition_by="event_date",
    cluster_by=["event_type", "traffic_source", "browser"],

```

Bảng này giữ event-level detail, phù hợp để audit/debug hoặc làm base cho các view dedup.

**9. Nhánh Aggregate 5 Phút**

Từ `enriched_events`, pipeline rẽ ra nhánh aggregate realtime.

Window logic:

```python
windowed_events = enriched_events.window(
    type=FixedWindow,
    size=5_minutes,
    allowed_lateness=600_seconds,
    trigger=AfterWatermark(
        early=AfterProcessingTime(60_seconds),
        late=AfterCount(1)
    ),
    accumulation_mode=ACCUMULATING
)
```

Key aggregate:

```python
aggregate_key = (
    event_date,
    traffic_source,
    browser,
    event_type,
    page_type,
)
```

Group:

```python
grouped = windowed_events.group_by_key(aggregate_key)
```

Build aggregate row:

```python
def create_aggregate_record(key, rows, window):
    session_ids = distinct(row["session_id"] for row in rows)
    user_ids = distinct(row["user_id"] for row in rows if row["user_id"] is not None)
    purchase_events = count(row for row in rows if row["is_conversion"])
    lags = [row["event_lag_seconds"] for row in rows]

    aggregate_id = concat(
        window.start,
        key.event_date,
        key.traffic_source,
        key.browser,
        key.event_type,
        key.page_type
    )

    return {
        "aggregate_id": aggregate_id,
        "window_start": window.start,
        "window_end": window.end,
        "event_date": key.event_date,
        "traffic_source": key.traffic_source,
        "browser": key.browser,
        "event_type": key.event_type,
        "page_type": key.page_type,
        "total_events": len(rows),
        "unique_sessions": len(session_ids),
        "unique_users": len(user_ids),
        "purchase_events": purchase_events,
        "avg_event_lag_seconds": avg(lags),
        "version_emitted_at": now_utc(),
    }
```

Ghi vào BigQuery:

```python
write_bigquery(
    table="events_5m",
    rows=aggregate_rows,
    partition_by="event_date",
    cluster_by=["traffic_source", "event_type", "page_type"],
    mode="append"
)
```

Vì dùng accumulating mode, cùng một `aggregate_id` có thể có nhiều version do early/late firing.

**10. Nhánh Sessionization**

Từ `enriched_events`, pipeline rẽ tiếp sang session metrics.

Key theo session:

```python
session_events = enriched_events.map(
    lambda row: (row["session_id"], row)
)
```

Session window:

```python
session_windowed = session_events.window(
    type=SessionWindow,
    gap=30_minutes,
    allowed_lateness=600_seconds,
    trigger=AfterWatermark(
        early=AfterProcessingTime(60_seconds),
        late=AfterCount(1)
    ),
    accumulation_mode=ACCUMULATING
)
```

Group theo session:

```python
grouped_sessions = session_windowed.group_by_key()
```

Build session metric:

```python
def build_session_metric(session_id, rows):
    rows = list(rows)

    if len(rows) > 5000:
        rows = rows[:5000]

    rows = sort_by(rows, "event_timestamp")

    if rows is empty:
        return None

    start = rows[0]["event_timestamp"]
    end = rows[-1]["event_timestamp"]

    cart_count = count(row for row in rows if row["event_type"] == "cart")
    purchase_count = count(row for row in rows if row["event_type"] == "purchase")

    category_counts = count_by(
        row["product_category"]
        for row in rows
        if row["product_category"] is not None
    )

    return {
        "session_record_id": session_id,
        "session_id": session_id,
        "user_id": rows[0]["user_id"],
        "traffic_source": rows[0]["traffic_source"],
        "browser": rows[0]["browser"],

        "session_start": start,
        "session_end": end,
        "session_duration_seconds": max(0, end - start),

        "event_count": len(rows),
        "pageview_count": count(
            row for row in rows
            if row["page_type"] in {"home", "department", "category", "product"}
        ),
        "product_view_count": count(
            row for row in rows
            if row["page_type"] == "product"
        ),
        "cart_count": cart_count,
        "purchase_count": purchase_count,

        "saw_home": any(row["page_type"] == "home" for row in rows),
        "saw_product": any(row["page_type"] == "product" for row in rows),
        "added_to_cart": cart_count > 0,
        "purchased": purchase_count > 0,

        "top_category": category_with_max_count(category_counts),
        "session_date": date(start),
        "version_emitted_at": now_utc(),
        "processed_at": now_utc(),
    }
```

Ghi vào BigQuery:

```python
write_bigquery(
    table="session_metrics",
    rows=session_metrics,
    partition_by="session_date",
    cluster_by=["traffic_source", "browser", "purchased"],
    mode="append"
)
```

Tương tự aggregate, session table cũng có thể có nhiều version cho cùng một `session_id`.

**11. Storage Layer**

Pipeline trực tiếp ghi 4 bảng chính:

```text
events_raw
events_deadletter
events_5m
session_metrics
```

Vai trò:

```text
events_raw
  Event-level enriched data, append-only.

events_deadletter
  Bad records, parse/validate errors.

events_5m
  Realtime aggregate theo fixed window 5 phút, append nhiều version.

session_metrics
  Session-level metrics, append nhiều version.
```

**12. Serving Layer**

Serving layer được tạo bằng BigQuery views để tránh double count và cung cấp bảng đọc ổn định hơn cho dashboard/mart.

View 1: `v_events_raw_dedup`

Logic:

```sql
SELECT *
FROM events_raw
QUALIFY ROW_NUMBER() OVER (
    PARTITION BY event_id
    ORDER BY ingested_at DESC, processing_time DESC
) = 1
```

Mục đích: lấy 1 bản mới nhất cho mỗi `event_id`.

View 2: `v_events_5m_latest`

Logic:

```sql
SELECT *
FROM events_5m
QUALIFY ROW_NUMBER() OVER (
    PARTITION BY aggregate_id
    ORDER BY version_emitted_at DESC
) = 1
```

Mục đích: vì aggregate có nhiều version, view này chỉ lấy bản mới nhất cho mỗi aggregate window/key.

View 3: `v_session_metrics_latest`

Logic:

```sql
SELECT *
FROM session_metrics
QUALIFY ROW_NUMBER() OVER (
    PARTITION BY session_id
    ORDER BY version_emitted_at DESC
) = 1
```

Mục đích: lấy session metric mới nhất cho mỗi session.

View 4: `v_session_funnel`

Nguồn: `v_session_metrics_latest`.

Logic:

```sql
SELECT
    session_id,
    user_id,
    traffic_source,
    browser,
    session_start AS session_started_at,
    saw_home,
    saw_product,
    added_to_cart,
    purchased,
    session_duration_seconds,
    event_count,
    top_category
FROM v_session_metrics_latest
```

Dùng để phân tích funnel theo session.

View 5: `v_daily_channel_kpis`

Nguồn: `v_events_raw_dedup`.

Logic:

```sql
SELECT
    event_date,
    traffic_source,
    COUNT(*) AS total_events,
    COUNT(DISTINCT session_id) AS total_sessions,
    COUNT(DISTINCT user_id) AS total_users,
    COUNTIF(event_type = 'purchase') AS purchase_events,
    SAFE_DIVIDE(
        COUNTIF(event_type = 'purchase'),
        COUNT(DISTINCT session_id)
    ) AS purchase_per_session
FROM v_events_raw_dedup
GROUP BY event_date, traffic_source
```

Dùng cho KPI theo ngày và channel.

View 6: `v_product_interest`

Nguồn: `v_events_raw_dedup`.

Logic:

```sql
SELECT
    event_date,
    product_id,
    product_name,
    product_category,
    product_department,
    COUNTIF(page_type = 'product') AS product_views,
    COUNTIF(event_type = 'cart') AS add_to_cart_events,
    COUNTIF(event_type = 'purchase') AS purchase_events
FROM v_events_raw_dedup
WHERE product_id IS NOT NULL
GROUP BY
    event_date,
    product_id,
    product_name,
    product_category,
    product_department
```

Dùng để phân tích interest/conversion theo sản phẩm.

**Tóm Tắt Luồng Abstract**

```python
messages = ReadFromPubSub(subscription)

valid_events, deadletters = messages | ParseValidate()

deadletters | WriteBigQuery("events_deadletter")

deduped_events = (
    valid_events
    | KeyBy(event_id)
    | StatefulDeduplicate(ttl=60_minutes)
)

product_map = LoadProductDimensionAtStartup()

enriched_events = (
    deduped_events
    | EnrichWithUriAndProduct(product_map)
)

enriched_events | WriteBigQuery("events_raw")

aggregates_5m = (
    enriched_events
    | FixedWindow(5_minutes, accumulating=True)
    | KeyBy(event_date, traffic_source, browser, event_type, page_type)
    | GroupByKey()
    | BuildAggregateMetrics()
)

aggregates_5m | WriteBigQuery("events_5m")

session_metrics = (
    enriched_events
    | KeyBy(session_id)
    | SessionWindow(gap=30_minutes, accumulating=True)
    | GroupByKey()
    | BuildSessionMetrics()
)

session_metrics | WriteBigQuery("session_metrics")
```

Serving:

```text
events_raw
  -> v_events_raw_dedup
  -> v_daily_channel_kpis
  -> v_product_interest

events_5m
  -> v_events_5m_latest

session_metrics
  -> v_session_metrics_latest
  -> v_session_funnel
```
